## 1. Eco real (Web Audio DelayNode + feedback)

**Antes:** UI placeholder sin backend.

**Ahora:** [`AudioEnginePro.ts`](../../Core/audio/AudioEnginePro.ts) tiene
`actualizarEco(intensidad, tiempoSeg)` con el patrón canónico:

```
mixBus → ecoDelay → ecoFeedback → (loop a ecoDelay)
              ↓
           ecoWet → output
```

Mapeo UI:
- **Intensidad 0–100** → `wet` hasta 0.7 + `feedback` hasta 0.6 (tope para
  evitar runaway en feedback infinito)
- **Tiempo 0–100** → `delayTime` 0.05–0.8 s (cubre slap-back, delay clásico
  y dub)

Cuando el toggle se apaga, intensidad=0 silencia wet+feedback y la ruta
vuelve al `directBus` (latencia mínima Android).

## 2. Distorsión real con 18 presets

**Antes:** UI placeholder con 3 opciones falsas.

**Ahora:** WaveShaperNode (oversample 4x) + 18 presets agrupados:

| Grupo | Presets |
|---|---|
| Cálidos / tube | Tubo Cálido, Tubo Cremoso, Vintage Drive, Lo-Fi Tape |
| Crunch | Crunch Clásico, Overdrive Blues, Rock 70s |
| Hard / Metal | Distorsión Dura, Heavy Metal, Thrash, Death Metal |
| Fuzz | Fuzz Big Muff, Fuzz Tone, Octave Fuzz |
| Experimentales | Bit Crusher, Megáfono, Teléfono, Wave Folder |

7 generadores de curva implementados:
- `tanh` — saturación cálida tipo tubo
- `tanh_asym` — tubo con asimetría (armónicos pares)
- `sigmoid_asym` — fuzz con offset DC (Big Muff)
- `hard_clip` — clipeo plano (metal)
- `fold` — wave folding (alien, FM-like)
- `bitcrush` — cuantización 4-bit (lo-fi)
- `octave_fuzz` — full-wave rectifier (octave up)

Cada preset = curva + drive + pre-EQ (color de entrada) + post-EQ (taming
de chillidos) + compensación de loudness.

**Bug corregido:** la curva `fold` salía fuera de rango ±1 con drive alto
(un solo rebote no convergía cuando |y|>3). Cambiado a loop iterativo
hasta convergencia.

API pública: `cargarPresetDistorsion(presetId)` + `actualizarDistorsion(
intensidad)`.

## 4. Pitos detrás del panel FX no se activan

**Bug:** con el panel abierto, deslizar el dedo desde un pito visible
hacia debajo del panel activaba pitos tapados — el cache de rects de
`usePointerAcordeon` no sabía que el panel los ocluía y el imán los
seleccionaba igual.

**Fix:** `obtenerRectsBloqueadores` (prop opcional del hook) +
`enRectBloqueador()` que hace early-return en `encontrarPosEnPunto` y en
los handlers `handleTouchStart` / `handlePointerDown`.

El simulador pasa `panel.getBoundingClientRect()` cuando `modales.efectos`
está abierto.

## 6. Audio activo con panel FX abierto

`desactivarAudio` ahora excluye explícitamente el modal `'efectos'`:

desactivando audio como antes. Solo el FX permite tocar simultáneamente.

## 7. Grabación admin: publicar como Canción Hero

**Componente nuevo:**
[`ModalGrabacionAdmin.tsx`](./Componentes/ModalGrabacionAdmin.tsx) +
`.css`. Solo se muestra si `usuario.rol === 'admin'`. Reemplaza al
`ModalGuardarSimulador` para admins.

### Funcionalidades

- **Resumen** de la grabación (notas, duración, BPM, tono)
- **Volver a grabar todo**: descarta la grabación pendiente y reinicia
  REC sin guardar (útil para corregir notas sucias)
- **Toggle Guardar como**:
  - 🟢 Mi grabación personal → `grabaciones_hero` (igual que alumno)
  - 🟡 Canción para alumnos → `canciones_hero` via `crearCancionV2`
- **Si Canción Hero**: campos extra (autor, dificultad, tipo, MP3 fondo
  opcional). El MP3 sube a bucket `pistas_hero` via `subirAudioFondoV2`.
- **Mobile-first**: pantalla casi completa <480px, controles grandes
  touch-friendly, padding-bottom 60-80px para holgura con teclado virtual

### Hook extendido

`useGrabacionProMax` ahora expone `guardarComoCancionHero(datos)` que:
1. Llama a `crearCancionV2()` con la secuencia capturada
2. Si hay MP3, sube a storage y actualiza `audio_fondo_url` con la URL pública
3. No aborta si falla solo el MP3 — la canción ya quedó guardada

`TipoGrabacionPendiente` ampliado a `'competencia' | 'practica_libre' |
'cancion_hero'`.

### Bugs corregidos en la implementación

**Bug 1: teclado virtual tapaba inputs en mobile landscape.**
- Fix: `onFocus` en cada input/select/textarea hace `scrollIntoView({
  block: 'center' })` con 300ms de delay (espera a que el teclado termine
  de aparecer)
- Modal usa `max-height: 92dvh` (dynamic viewport height — se adapta al
  teclado) con fallback `vh` para navegadores antiguos
- Media query landscape mobile específica con padding-bottom 80px

**Bug 2: metrónomo siempre se guardaba como `usoMetronomo: false`.**
- Causa: leíamos `metronomoEnRecRef.current` que se ponía a `null` en
  `detenerGrabacion` ANTES de que el modal apareciera
- Fix: leer `grabacion.grabacionPendiente?.metadata?.metronomo` que sí
  persiste hasta que se cierre el modal

**Bug 3: admin no podía guardar como "Mi grabación personal".**
- Causa: incompatibilidad de firmas. `ModalGuardarSimulador.onGuardar`
  espera `(titulo, descripcion)` posicional. Mi `ModalGrabacionAdmin.
  onGuardarPersonal` esperaba un objeto `{ titulo, descripcion }`.
  Al pasar `guardarPracticaLibre` (que es posicional) a un slot que
  llama con objeto, `titulo` recibía el objeto entero y `datos.titulo.
  trim()` fallaba con TypeError.
- Fix: alineado el contrato de `ModalGrabacionAdmin.onGuardarPersonal` a
  `(titulo: string, descripcion: string)` — mismo que `ModalGuardarSimulador`.

## Cómo retomar

1. Levantar dev: `npm run dev`
2. Login como admin
3. `/simulador-app` → REC → grabar → STOP → modal admin aparece
4. Probar:
   - "Volver a grabar todo" descarta y reinicia
   - "Mi grabación" guarda en `grabaciones_hero` (mis grabaciones)
   - "Canción para alumnos" guarda en `canciones_hero` (visible en
     `/acordeon-pro-max/admin/`)
   - Subir MP3 opcional
5. Para forzar admin sin tocar BD:
   ```js
   const u = JSON.parse(localStorage.getItem('usuario_actual'));
   u.rol = 'admin'; localStorage.setItem('usuario_actual', JSON.stringify(u));
   location.reload();
   ```
