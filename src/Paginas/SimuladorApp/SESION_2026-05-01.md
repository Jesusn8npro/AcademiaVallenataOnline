### `Juego/`
- **`JuegoSimuladorApp.tsx` + `.css`** — Orquestador del modo juego. Reemplaza al
  `SimuladorAppNormal` cuando hay `juegoActivo`. Reusa `useLogicaProMax` (toda la
  lógica de scoring/scheduler ya probada en producción) + `usePointerAcordeon`
  conectado a `hero.logica` para los inputs táctiles. Renderiza los pitos del
  SimuladorApp con `renderHilera`, monta `MenuPausaProMax`, `PantallaResultados`
  y `PantallaGameOverProMax` según el `estadoJuego`.
- **`PistaNotasVertical.tsx` + `.css`** — Visualización custom de notas cayendo:
  10 columnas, posición X por `botonId`, posición Y por progreso del tick.
  Color azul (`COLOR_ABRIENDO`) si el fuelle es abriendo, rojo (`COLOR_CERRANDO`)
  si es cerrando. `position: fixed` con `pointer-events: none` para no bloquear
  los toques al acordeón debajo.
- **`HeaderJuegoSimulador.tsx` + `.css`** — Header mobile-first del juego con
  pausa / salir / título canción / vida (solo competitivo) / combo / score.

## Decisiones técnicas clave

1. **No se duplica lógica de juego.** `useLogicaProMax` (50 KB) se reusa tal cual.
   El `JuegoSimuladorApp` solo es un "wrapper" que renderiza otro layout visual
   (vertical mobile en lugar del horizontal MAESTRO/ALUMNO de ProMax).

2. **Inicialización del juego** — `iniciarJuego(cancion, false, modoForzado)` se
   llama con el modo como **tercer parámetro forzado** porque `setModoPractica`
   actualiza estado React (async) y `iniciarJuego` lee de un ref interno que
   puede no estar actualizado en el mismo tick. Bug detectado y corregido en
   esta sesión.

3. **Coordinación de desmontaje** — al hacer EMPEZAR, primero se cierra el
   modal de configuración (`setCancionConfigurando(null)`) y con `setTimeout(30ms)`
   se cambia a `juegoActivo`. Sin ese delay, React/framer-motion fallaba con
   `removeChild` al desmontar dos sub-trees simultáneamente.

4. **Resaltado de pito objetivo** — se calcula vía `useMemo` y se pasa como
   prop al render de cada pito (no via `document.querySelector`). El regex
   normaliza `"A1-halar"` y `"A-1-halar"` al formato `"A-1"` para coincidir
   con el `data-pos` del pito en el DOM.

5. **Pista de notas en `position: fixed` encima del acordeón** — primero se
   intentó un layout flex column con zona-notas + acordeón sibling, pero el
   `.simulador-canvas` tiene `height: calc(...)` con `!important` que en
   landscape mobile colapsaba la zona-notas a casi 0. Approach final: pista
   absolute encima de TODO con `pointer-events: none` (estilo Synthesia).
   No se modifican las dimensiones del acordeón.

## Mapeo de modos (ConfigCancion → useLogicaProMax)

| ConfigCancion    | useLogicaProMax | Características                          |
|------------------|-----------------|------------------------------------------|
| `competitivo`    | `'ninguno'`     | Puntos, vida, combo, game over           |
| `libre`          | `'libre'`       | Puntos y combo, sin vida                 |
| `synthesia`      | `'synthesia'`   | Pausa en cada nota                       |
| `maestro_solo`   | `'maestro_solo'`| Rebobina, BPM, sin scoring               |

Colores del fuelle (de `TiposProMax.ts`):
- 🔵 Abriendo (halar) = `COLOR_ABRIENDO` `#3b82f6`
- 🔴 Cerrando (empujar) = `COLOR_CERRANDO` `#ef4444`
